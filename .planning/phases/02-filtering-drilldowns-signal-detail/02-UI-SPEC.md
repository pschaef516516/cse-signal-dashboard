## Interaction Contract

### Filter Pills

- Clicking a pill updates per-tab filter state in App.jsx (`churnFilter` / `enrollmentFilter`)
- Filter applies to: all stat cards on the active tab, all charts on the active tab
- "Posts Ingested" stat card: filter by `captured_date` field (not `created_at`) when filter is not "All"
- Switching tabs does NOT reset either filter (D-03)
- Default on load: "All" selected for both tabs

### Clickable Charts

Clicking a chart element calls a handler that sets drawer state:

```
drawerOpen: true
drawerFilter: { field: 'source' | 'week' | 'severity', value: <clicked value> }
drawerSignals: <pre-filtered array>
```

Clickable elements (D-05):
- CommunityChart: click on a Bar/Cell â€” filter by `source`
- SignalVolumeChart: click on a Bar â€” filter by `week`
- SeverityChart: click on a Bar/Cell â€” filter by `severity`

NOT clickable: MatchRateChart, ConfidenceHistogram, PostsVsSignalsChart

Cursor on clickable charts: `cursor: pointer` applied at chart wrapper level.

### Drawer Navigation

1. Chart click â†’ drawer opens with signal list
2. Signal card click â†’ signal detail replaces list (no URL change)
3. "Back to signals" click â†’ signal list reappears
4. Backdrop click â†’ drawer closes, state resets to `drawerOpen: false`

Drawer state resets fully on close â€” reopening always shows the signal list, not the last viewed detail.

### Animations

- Drawer open/close: `transition: transform 0.25s ease` â€” do not use `visibility` or `display` toggling (breaks transition)
- Filter pill active state change: `transition: background 0.15s, color 0.15s, border-color 0.15s` â€” matches existing tab transition pattern
